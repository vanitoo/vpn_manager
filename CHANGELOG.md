# Changelog

Все значимые изменения Warp Private Network фиксируются в этом файле.

Формат основан на Keep a Changelog. До стабильного релиза `1.0` версии описывают этапы развития проекта, а не совместимость публичного API.

## [Unreleased]

### Added

- Модульная система поддержки в `app/support/`.
- Telegram Topics HelpDesk: отдельная тема для каждого обращения.
- Таблицы `support_tickets` и `support_messages`.
- Автоматическая карточка пользователя и VPN-подписки для оператора.
- Передача текста, фото, видео, документов, аудио, голосовых и стикеров между пользователем и оператором.
- Команды операторов `/take`, `/wait`, `/close`.
- Команда `/chatid` для получения ID группы поддержки.
- Статистика обращений в административном меню.
- Настройки `SUPPORT_ENABLED` и `SUPPORT_GROUP_ID`.

### Changed

- Пользовательский раздел `Поддержка` переименован в `❓ Помощь`.
- Сценарий помощи теперь ведёт пользователя через FAQ и диагностику до создания тикета.
- Telegram используется как операторский интерфейс, а SQLite остаётся источником данных.
- Полностью переписана документация проекта.
- Удалены все упоминания и история исходного книжного бота.

## [0.7.0] - 2026-07-18

### Added

- Пользовательский раздел помощи.
- Выбор категории проблемы перед обращением:
  - не подключается;
  - проблема с оплатой;
  - потеряна ссылка;
  - низкая скорость;
  - другое.
- Автоматическая диагностика локальной подписки и данных Remnawave.
- Возможность закрыть обращение пользователем.
- Административная статистика поддержки.

### Changed

- FAQ, диагностика и связь со специалистом объединены в единый пользовательский сценарий.
- Поддержка выделена в автономный модуль для возможного будущего переноса в отдельный сервис.

## [0.6.0] - 2026-07-17

### Added

- Хранилище FAQ в SQLite.
- Пользовательский список частых вопросов.
- Просмотр ответа на отдельный вопрос.
- Управление FAQ из админки:
  - создание;
  - изменение вопроса;
  - изменение ответа;
  - скрытие и публикация;
  - удаление.

## [0.5.0] - 2026-07-16

### Added

- Модульные внешние платёжные провайдеры.
- Telegram Stars.
- ЮKassa с созданием платёжной ссылки и ручной проверкой статуса без обязательного webhook.
- Cryptomus.
- Конфигурационные заготовки для Lava и Platega.
- Сохранение контакта для формирования чеков.
- TTL ожидающих платежей.
- Защита запуска от неполной конфигурации отключённых платёжных провайдеров.

### Changed

- Стоимость тарифов хранится в рублях.
- Количество Telegram Stars рассчитывается по `STARS_RUB_PER_STAR`.
- Доступ выдаётся только после подтверждения успешной оплаты.

## [0.4.0] - 2026-07-15

### Added

- Административное меню.
- Управление пользователями.
- Управление тарифами.
- Просмотр Remnawave users, nodes и squads.
- Синхронизация данных Remnawave с локальной базой.
- Выдача тестового доступа.
- Блокировка и активация пользователей.
- Служебные тарифы, скрытые из пользовательского каталога.

## [0.3.0] - 2026-07-14

### Added

- Интеграция с Remnawave 2.7.x.
- Поиск пользователя по служебному email.
- Создание пользователя Remnawave.
- Продление действующей подписки.
- Назначение internal squad.
- Поддержка external squad.
- Ограничения трафика и количества устройств.
- Формирование ссылки подписки по `subscriptionUrl`, `shortUuid` или `uuid`.
- Диагностические запросы к API Remnawave.
- Поддержка nginx cookie и Basic Auth перед Remnawave.
- Локальный безопасный stub-режим при отсутствии API.

### Changed

- При активной подписке новые дни добавляются к текущей дате окончания.
- При истёкшей подписке новый период начинается с текущего момента.

## [0.2.0] - 2026-07-13

### Added

- SQLite-хранилище пользователей, тарифов, платежей и подписок.
- Главное пользовательское меню.
- Каталог тарифов.
- Раздел `Мой VPN`.
- Персональные ссылки подписок.
- Продление доступа.
- Тестовый период.
- Резервное копирование базы.
- Ротация логов.
- Уведомления администраторам о запуске, остановке и покупках.
- Рассылки и правила напоминаний.

## [0.1.0] - 2026-07-12

### Added

- Начальная версия Telegram VPN-бота на aiogram.
- Конфигурация через `.env`.
- Работа в режиме polling без обязательного сервера и webhook.
- Поддержка прокси для Telegram Bot API.
- Команды `/start`, `/cancel`, `/admin` и `/backup`.
- Базовая структура проекта `vpn_manager`.
