# Roadmap

План развития Warp Private Network. Порядок может меняться по результатам эксплуатации и обратной связи.

## 0.8 — Support hardening

- приоритеты обращений;
- назначение ответственного оператора;
- поиск и фильтрация тикетов;
- повторное открытие закрытого обращения;
- шаблоны быстрых ответов;
- расширенная диагностика nodes и устройств;
- защита от дублирующих обращений;
- метрики времени первого ответа и закрытия.

## 0.9 — Commercial tools

- промокоды;
- реферальная система;
- гибкие пробные периоды;
- скидки и индивидуальные тарифы;
- история действий администратора;
- улучшенная аналитика платежей и продлений;
- уведомления о неуспешных платежах.

## 1.0 — Stable release

- зафиксированная схема конфигурации;
- миграции базы данных;
- автоматические тесты основных сценариев;
- проверка обновления с предыдущих версий;
- стабильная документация установки и эксплуатации;
- подготовленный процесс релизов и отката;
- аудит безопасности.

## После 1.0

- PostgreSQL;
- несколько экземпляров бота;
- REST API;
- отдельный операторский интерфейс при необходимости;
- дополнительные VPN-протоколы и панели;
- мультиязычный интерфейс;
- расширенная observability: метрики, трассировка и алерты.

## Не входит в ближайший план

- хранение закрытых ключей пользователей;
- собственная реализация VPN-протоколов;
- замена Remnawave полноценной панелью управления;
- обязательный публичный веб-интерфейс для клиентов.
