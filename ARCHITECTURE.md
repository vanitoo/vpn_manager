# Architecture

## Назначение

Warp Private Network управляет жизненным циклом VPN-подписки через Telegram: регистрацией пользователя, выбором тарифа, оплатой, выдачей доступа, продлением, диагностикой и поддержкой.

## Общая схема

```text
Telegram User
     │
     ▼
 aiogram routers + FSM
     │
 ┌───┼───────────────┬──────────────┐
 │   │               │              │
 ▼   ▼               ▼              ▼
VPN  Payments        FAQ          Support
 │   │               │              │
 └───┴───────┬───────┴──────────────┘
             ▼
           SQLite
             │
             ▼
       Remnawave Client
             │
      ┌──────┴──────┐
      ▼             ▼
    Squads         Nodes
```

## Слои

### Telegram transport

Обработчики aiogram принимают команды, callback-запросы и сообщения. Они не должны хранить состояние бизнеса вне FSM и базы данных.

### Application logic

Сценарии покупки, продления, выдачи доступа, диагностики и поддержки координируют работу хранилища, платёжных провайдеров и Remnawave.

### Integrations

- `app/remnawave.py` — Remnawave API;
- платёжные модули — Stars и внешние провайдеры;
- Telegram Bot API — пользовательский и операторский интерфейс.

### Persistence

SQLite хранит пользователей, тарифы, платежи, подписки, FAQ, рассылки и тикеты. Telegram Topics не считается постоянным хранилищем.

## Ключевые модули

```text
app/main.py
```

Инициализация настроек, базы, логирования, команд, routers и polling.

```text
app/config.py
```

Общие параметры приложения и интеграций.

```text
app/db.py
```

Основные операции с пользователями, тарифами, платежами и подписками.

```text
app/remnawave.py
```

Изолированный HTTP-клиент Remnawave. Создаёт и продлевает пользователей, получает squads и выполняет диагностику.

```text
app/support/
```

Автономный модуль помощи:

- `settings.py` — конфигурация группы;
- `storage.py` — тикеты и сообщения;
- `diagnostics.py` — проверки пользователя;
- `user.py` — пользовательский сценарий;
- `operator.py` — мост Telegram Topics;
- `routes.py` — объединение routers.

## Основные потоки

### Покупка

```text
Выбор тарифа
  → выбор провайдера
  → создание платежа
  → подтверждение оплаты
  → создание/продление Remnawave user
  → сохранение подписки
  → выдача ссылки
```

### Продление

```text
Локальная подписка
  → поиск пользователя Remnawave
  → определение базовой даты
  → обновление expireAt
  → обновление SQLite
```

### Поддержка

```text
FAQ
  → диагностика
  → категория
  → тикет SQLite
  → Telegram topic
  → сообщения в обе стороны
  → закрытие тикета и темы
```

## Источники истины

- подписки и платежи: SQLite плюс подтверждённое состояние провайдера;
- VPN user и параметры доступа: Remnawave;
- тикеты и история поддержки: SQLite;
- Telegram Topics: операторский интерфейс;
- конфигурация: environment variables.

## Ошибки и отказоустойчивость

- отключённый провайдер не должен мешать запуску;
- сетевые ошибки записываются в лог;
- Remnawave stub допускается только как безопасный режим разработки;
- сообщения поддержки сохраняются вместе с направлением и идентификаторами Telegram;
- резервные копии SQLite должны создаваться вне основного файла базы.

## Ограничения текущей версии

- один процесс приложения;
- SQLite;
- polling;
- нет публичного REST API;
- нет распределённой блокировки операций;
- Telegram Topics зависят от доступности Telegram Bot API.
