# Support

## Обзор

Система поддержки встроена в основной Telegram-бот, но реализована отдельным модулем `app/support/`. Пользователь общается с ботом в личном чате, оператор отвечает через Telegram-группу с включёнными темами.

## Пользовательский сценарий

```text
❓ Помощь
  ├─ 📖 FAQ
  ├─ 🩺 Диагностика
  └─ 💬 Написать специалисту
```

Перед созданием обращения пользователь выбирает категорию:

- не подключается;
- проблема с оплатой;
- потеряна ссылка;
- низкая скорость;
- другое.

После этого бот выполняет диагностику. Если проблема не решена, пользователь отправляет первое сообщение и создаётся тикет.

## Настройка Telegram-группы

1. Создайте отдельную Telegram-группу.
2. Включите режим `Темы`.
3. Добавьте бота администратором.
4. Разрешите боту отправлять сообщения и управлять темами.
5. Отправьте в группе `/chatid`.
6. Добавьте полученный ID в `.env`.

```env
SUPPORT_ENABLED=true
SUPPORT_GROUP_ID=-1001234567890
```

После изменения конфигурации перезапустите приложение.

## Как создаётся тикет

1. В SQLite создаётся запись `support_tickets`.
2. Бот создаёт forum topic с номером тикета и именем пользователя.
3. В тему отправляется карточка пользователя и результат диагностики.
4. Сообщение пользователя копируется в тему.
5. Связь между тикетом и `message_thread_id` сохраняется в базе.

## Карточка оператора

Карточка может включать:

- номер и категорию обращения;
- Telegram ID;
- имя и username;
- локальный статус подписки;
- тариф и дату окончания;
- ссылку подписки;
- Remnawave UUID;
- squad;
- историю и сумму платежей;
- результат автоматической диагностики.

Поля, которых нет в API или локальной базе, не выдумываются и не показываются как достоверные.

## Ответ оператора

Оператор пишет обычное сообщение внутри темы. Бот копирует его пользователю в личный чат и записывает событие в `support_messages`.

Поддерживаются:

- текст;
- фото;
- видео;
- документы;
- голосовые сообщения;
- аудио;
- анимации;
- стикеры.

Служебные команды темы:

```text
/take   статус in_progress
/wait   статус waiting_user
/close  статус closed и закрытие темы
```

## Статусы

```text
new
in_progress
waiting_user
closed
```

`new` создаётся при первом сообщении пользователя. Оператор переводит обращение в работу командой `/take` или обычным ответом. `/wait` означает ожидание пользователя. `/close` завершает обращение.

Пользователь также может закрыть собственное обращение кнопкой в личном чате.

## Хранение данных

### support_tickets

Хранит тикет, пользователя, категорию, статус, Telegram topic, назначенного оператора и временные метки.

### support_messages

Хранит направление сообщения, идентификаторы сообщений в обоих чатах, тип вложения и короткое текстовое представление.

SQLite является источником истины. Удаление темы Telegram не должно удалять историю тикета.

## Диагностика

Автоматические проверки могут включать:

- наличие пользователя в локальной базе;
- существование подписки;
- срок действия;
- наличие subscription URL;
- идентификатор Remnawave;
- данные тарифа;
- платежную историю;
- доступность настроенной интеграции Remnawave.

## Работа операторов

- отвечайте только внутри соответствующей темы;
- не пересылайте пользователям внутренние токены и диагностические секреты;
- используйте `/take`, чтобы другие операторы видели, что обращение уже взято;
- используйте `/wait`, если требуется ответ клиента;
- закрывайте завершённые обращения командой `/close`;
- не удаляйте темы вместо штатного закрытия тикета.

## Диагностика проблем

Если тема не создаётся:

- проверьте `SUPPORT_GROUP_ID`;
- убедитесь, что это supergroup с включёнными темами;
- проверьте права администратора у бота;
- убедитесь, что бот добавлен именно в указанную группу;
- изучите `logs/bot.log`.

Если ответ не доставляется пользователю:

- пользователь мог заблокировать бота;
- тема может не соответствовать тикету;
- сообщение может содержать неподдерживаемый тип;
- Telegram API может временно быть недоступен.
